AddCustomReadWriteAsyncRepository<TRepositoryInterface, TRepository, TEntity, TId>(IServiceCollection, Func<IServiceCollection, Type, Type, IServiceCollection>)

Registers a mapping of read / write async repository interfaces to custom repository type in the service collection.

Declaration
public static IServiceCollection AddCustomReadWriteAsyncRepository<TRepositoryInterface, TRepository, TEntity, TId>(this IServiceCollection serviceCollection, Func<IServiceCollection, Type, Type, IServiceCollection> registrationFunction)
    where TRepositoryInterface : class, IReadWriteRepositoryAsync<TEntity, TId> where TRepository : class, TRepositoryInterface, IReadWriteRepositoryAsync<TEntity, TId> where TEntity : class, IHasId<TId>
Parameters
Type Name Description
IServiceCollection serviceCollection

The service collection to add the repositories to.
Func<IServiceCollection, Type, Type, IServiceCollection> registrationFunction

The method used for registration.
Returns
Type Description
IServiceCollection

The same passed in serviceCollection used for method chaining.
Type Parameters
Name Description
TRepositoryInterface

The interface type of the custom repository.
TRepository

The implementation type of the custom repository.
TEntity

The entity type.
TId

The entity Id property type.
Remarks

This method registers the following mappings:

  • TRepositoryInterface to TRepository
  • IReadWriteRepositoryAsync<TEntity, TId> to TRepository
  • IWriteRepositoryAsync<TEntity, TId> to TRepository
  • IReadRepositoryAsync<TEntity, TId> to TRepository
It allows to specify how the TRepository is registered in the service collection using the registrationFunction parameter. Example below shows how to register the custom repository type using the AddScoped extension method: 
services.AddCustomAsyncRepository<ICarReadWriteRepository, CarReadWriteRepository, Car, int>
        ((collection,
        sourceType, targetType) => collection.AddScoped(sourceType, targetType));
